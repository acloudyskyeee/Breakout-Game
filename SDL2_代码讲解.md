# SDL2 打砖块项目说明文档

这份文档基于仓库中的 `breakout.cpp`，面向**刚接触 SDL2** 的同学，解释这个小游戏的整体框架和各部分代码职责。

---

## 1. 项目整体框架（先看全局）

这个游戏可以抽象成 5 个模块：

1. **初始化模块**：初始化 SDL、创建窗口和渲染器。
2. **事件模块**：处理关闭窗口、按键按下等离散事件。
3. **更新模块**：每一帧根据 `dt` 更新挡板、小球、碰撞、胜负状态。
4. **渲染模块**：把砖块、挡板、小球画到屏幕上。
5. **主循环模块**：重复执行“事件 -> 更新 -> 渲染”，直到退出。

代码中这些都集中在 `Game` 结构体里，`main()` 负责驱动整个生命周期。

---

## 2. 数据结构：游戏对象都放在哪里

### 2.1 Brick（砖块）

```cpp
struct Brick {
    SDL_Rect rect{};
    bool alive = true;
    SDL_Color color{255, 255, 255, 255};
};
```

含义：

- `rect`：砖块的位置和尺寸（整型矩形）。
- `alive`：砖块是否还存在（被击中后会设为 `false`）。
- `color`：砖块颜色。

### 2.2 Game（游戏状态与行为）

`Game` 是核心对象，包含：

- **常量配置**：窗口大小、砖块行列、挡板尺寸、球速等。
- **SDL 资源**：`SDL_Window* window`、`SDL_Renderer* renderer`。
- **游戏状态**：砖块数组、挡板和小球位置、速度、生命值、分数、胜负标记。
- **行为函数**：初始化、事件处理、更新、渲染、重开等。

可以把它理解成一个“迷你游戏引擎实例”。

---

## 3. 初始化流程（init）

`init()` 的流程非常典型：

1. `SDL_Init(SDL_INIT_VIDEO)`：初始化 SDL 视频子系统。
2. `SDL_CreateWindow(...)`：创建窗口。
3. `SDL_CreateRenderer(...)`：创建渲染器。
4. `createBricks()`：生成关卡砖块。
5. `resetBall(false)`：放置初始球并设置初速度。

如果任一步失败，会打印 `SDL_GetError()` 并返回 `false`。

> 初学建议：SDL 项目里“先看 init + main loop + shutdown”通常就能快速理解架构。

---

## 4. 资源释放（shutdown）

`shutdown()` 依次释放：

1. `SDL_DestroyRenderer(renderer)`
2. `SDL_DestroyWindow(window)`
3. `SDL_Quit()`

这是 SDL 程序很重要的收尾步骤，避免资源泄漏。

---

## 5. 关卡与状态重置

### 5.1 createBricks()

功能：按行列生成砖块并赋色。

- 通过 `kBrickRows / kBrickCols` 双循环创建砖块。
- 利用 `startX` 让砖块墙在窗口中水平居中。
- 每行从调色板拿颜色，得到清晰层次。

### 5.2 resetBall(bool randomDirection)

功能：把球放回初始位置并设置速度方向。

- 球回到窗口下方中间。
- `randomDirection=true` 时，随机左右方向，避免每次复活轨迹完全一致。

### 5.3 restart()

功能：重开一局，恢复关键状态：

- 生命值、分数、胜负状态。
- 挡板位置、砖块布局、球速度。

---

## 6. 输入与事件：SDL 两类常见读取方式

这个项目同时演示了 SDL 的两种输入方式：

1. **事件队列（离散事件）**：`SDL_PollEvent`
   - 用来处理 `SDL_QUIT`、按下 `ESC`、`R` 重开等“瞬时事件”。

2. **键盘状态（持续输入）**：`SDL_GetKeyboardState`
   - 用来处理左右移动挡板（按住持续移动）。

这也是 SDL 游戏开发中最常见的组合。

---

## 7. 每帧更新（update）在做什么

`update(float dt)` 是游戏逻辑核心，按顺序大致是：

1. **若游戏已结束则直接返回**。
2. **读取键盘状态**，更新挡板横向位置并 `clamp` 到边界。
3. **按速度积分更新球位置**：`ball.x += vx * dt; ball.y += vy * dt;`
4. **墙体碰撞**：
   - 碰左/右墙：翻转 `vx`
   - 碰顶墙：翻转 `vy`
5. **挡板碰撞**：
   - 用 `SDL_HasIntersection` 判断。
   - 根据球击中挡板的位置计算反弹角度（不是固定反弹，手感更好）。
6. **砖块碰撞**：
   - 击中后砖块设为 `alive=false`，分数 +10。
   - 根据球心和砖块中心相对位置，决定翻转 `vx` 还是 `vy`。
7. **掉球判定**：
   - 球掉出底部则扣命。
   - 命归零 -> 失败；否则重置球。
8. **胜利判定**：
   - `bricksLeft() == 0` -> 通关。

### 为什么要用 `dt`（Delta Time）？

不同机器帧率不同，使用 `dt` 能让移动速度近似“与帧率无关”。

## 8. 渲染流程（render）

`render()` 的顺序：

1. 设置背景色并清屏 `SDL_RenderClear`。
2. 遍历绘制所有活着的砖块。
3. 绘制挡板。
4. 绘制小球。
5. `SDL_RenderPresent` 提交到屏幕。

这是一种典型的“每帧全量重绘”2D 渲染模式，结构清晰、易维护。

---

## 9. 主循环（main）如何串起一切

`main()` 中关键步骤：

1. 创建 `Game game`。
2. 调 `game.init()`。
3. 用 `SDL_GetPerformanceCounter()` 计算高精度帧间隔 `dt`。
4. 在 `while (!quit)` 中循环执行：
   - `game.handleEvents(quit)`
   - `game.update(dt)`
   - `game.render()`
5. 退出时 `game.shutdown()`。

并且有一行：

```cpp
dt = std::min(dt, 0.033f);
```

含义是限制最大 `dt`（约 30 FPS 对应帧时长），避免因为卡顿导致单帧时间过大，引发物理跳变。

## 10. 你可以按这个顺序阅读源码

如果你 SDL2 还不熟，建议读代码时按下面顺序：

1. `main()`（理解主循环）
2. `init()` / `shutdown()`（理解生命周期）
3. `handleEvents()`（理解输入入口）
4. `update()`（理解游戏规则）
5. `render()`（理解如何把状态画出来）
6. `createBricks()` / `resetBall()` / `restart()`（理解状态管理）

## 11. 新手常见问题

### Q1：为什么有时用 `SDL_FRect`，有时用 `SDL_Rect`？

- `SDL_FRect`（浮点）适合做平滑运动与速度积分。
- `SDL_Rect`（整型）适合渲染 API 和碰撞 API。

本项目里就先用浮点保存真实位置，碰撞/绘制时临时转整型。

### Q2：为什么既有 `SDL_KEYDOWN` 又有 `SDL_GetKeyboardState`？

- `KEYDOWN` 适合“按下一次触发一次”的动作（比如 `R` 重开）。
- `GetKeyboardState` 适合“按住持续生效”的动作（比如挡板移动）。

### Q3：窗口标题为什么会变化？

代码用 `updateWindowTitle()` 把分数、生命值和胜负状态写到窗口标题里，属于最轻量的 HUD 展示方式。
